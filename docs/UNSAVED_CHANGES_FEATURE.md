# 未保存数据提示功能

## 功能概述

为了防止用户忘记保存数据到数据库，系统现在会在有未保存更改时提供多重提示。

## 功能特性

### 1. 视觉提示指示器

- **位置**: 屏幕右下角固定浮动提示框
- **显示条件**: 
  - 仅在编辑模式（Edit Mode）下显示
  - 有未保存的更改时自动出现
- **显示内容**:
  - 警告图标（带动画效果）
  - 未保存更改的数量
  - "立即保存"按钮

### 2. 项目切换按钮徽章

- **位置**: 顶部导航栏的"Cloud Registry"项目选择按钮
- **显示**: 橙色数字徽章，显示未保存更改的数量
- **动画**: 脉冲动画效果，引起用户注意

### 3. 浏览器关闭/刷新警告

- **触发条件**: 用户尝试关闭标签页、刷新页面或离开网站时
- **行为**: 浏览器原生警告对话框，提示有未保存的更改
- **消息**: "您有未保存的更改，确定要离开吗？"

### 4. 项目切换保护

- **触发条件**: 用户尝试切换到其他项目时
- **行为**: 
  1. 首先询问是否保存当前项目
  2. 如果选择"确定"，先保存再切换
  3. 如果选择"取消"，再次确认是否丢弃更改
  4. 只有明确确认后才允许切换

## 技术实现

### Store 状态追踪

在 `store.ts` 中添加了两个新方法：

```typescript
hasUnsavedChanges: () => boolean
getUnsavedChangesCount: () => number
```

这些方法基于现有的脏数据追踪机制：
- `_dirtyDrawingIds`: 修改过的图纸ID集合
- `_deletedDrawingIds`: 删除的图纸ID集合
- `_dirtyConf`: 配置是否修改
- `_dirtyTracker`: 追踪器是否修改

### 组件结构

**UnsavedChangesIndicator.tsx**
- 独立的React组件
- 使用 `useEffect` 监听浏览器关闭事件
- 响应式设计，适配移动端

### 用户体验优化

1. **非侵入式**: 提示框位于右下角，不遮挡主要工作区域
2. **渐进式警告**: 从视觉提示到交互确认，层层保护
3. **快速保存**: 一键保存按钮，无需导航到其他页面
4. **状态同步**: 保存后提示自动消失

## 使用场景

### 场景1: 编辑图纸后忘记保存
1. 用户修改了图纸状态或添加了备注
2. 右下角立即出现橙色提示框
3. 项目切换按钮显示徽章
4. 用户点击"立即保存"按钮完成保存

### 场景2: 尝试切换项目
1. 用户在有未保存更改时点击项目选择器
2. 系统弹出确认对话框
3. 用户选择保存或丢弃更改
4. 根据选择执行相应操作

### 场景3: 意外关闭浏览器
1. 用户不小心点击关闭标签页
2. 浏览器显示原生警告
3. 用户可以取消关闭，返回保存数据

## 配置选项

目前功能默认启用，无需额外配置。

未来可扩展的配置项：
- 自动保存间隔（已有 `autoSyncInterval`）
- 提示框位置
- 警告阈值（多少更改后才提示）

## 注意事项

1. **只读模式**: 在只读模式下不显示提示（因为无法编辑）
2. **自动同步**: 如果启用了自动同步，提示会在同步后自动消失
3. **网络状态**: 保存失败时提示不会消失，用户可以重试

## 测试建议

1. 修改图纸数据，验证提示出现
2. 点击保存按钮，验证提示消失
3. 尝试切换项目，验证警告对话框
4. 尝试关闭浏览器，验证浏览器警告
5. 在只读模式下验证提示不出现
